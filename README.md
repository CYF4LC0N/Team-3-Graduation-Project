# Web Application Penetration Testing - DEPI Final Project

## Project Overview

This project is a practical web application penetration testing simulation conducted as part of DEPI Training.

Instead of developing a custom vulnerable application, this project utilizes an existing intentionally vulnerable lab environment to better simulate real-world penetration testing scenarios.

The assessment is performed on a pre-built lab designed for security training, allowing the team to focus on identifying, exploiting, and documenting real-world vulnerabilities in a controlled environment.

---

## Target Application

This assessment is conducted on the following vulnerable lab:

**WAPT Lab (Web Application Penetration Testing Lab)**  
https://github.com/selimwdev/WAPTLab

### Description
WAPT Lab is an intentionally vulnerable web application designed for penetration testing practice. It contains multiple common web vulnerabilities such as:

- Broken Access Control  
- Insecure Direct Object References (IDOR)  
- Input Validation Issues  
- Authentication & Session Weaknesses  
- File Handling Vulnerabilities  

The lab simulates real-world scenarios where testers interact with an existing application rather than building one from scratch.

---

## Project Objectives

- Perform a full penetration test on an existing vulnerable web application  
- Follow a structured, industry-standard testing methodology  
- Identify and exploit security vulnerabilities  
- Document vulnerabilities with Proof of Concept (PoC)  
- Deliver a professional security assessment report  

---

## Scope

The assessment is limited to the provided lab environment.

### In Scope:
- Authentication mechanisms  
- Authorization and access control  
- Input validation  
- Session management  
- File upload functionality  

### Out of Scope:
- Denial of Service attacks  
- Third-party services  
- Infrastructure outside the lab environment  

---

## Methodology

The penetration testing process follows a structured approach based on widely accepted practices (aligned with OWASP methodology):

1. Reconnaissance  
2. Enumeration  
3. Vulnerability Identification  
4. Exploitation  
5. Post-Exploitation Analysis  
6. Reporting and Remediation  

---

## Expected Vulnerabilities

Based on the lab design, the application may include (but is not limited to):

- SQL Injection  
- Cross-Site Scripting (XSS)  
- Broken Authentication  
- Insecure Direct Object Reference (IDOR)  
- File Upload Vulnerabilities  
- Broken Access Control
